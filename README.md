# Transaction Lab

Projeto simples para fins acadêmicos. Serve como base para o estudo de funcionalidades e tecnologias de backend.

## Tecnologias em estudo
* **Error handling:** Tratamento de exceções e respostas de erro.
* **Git:** Fluxo de trabalho e gestão de histórico.
* **Zod:** Validação de esquemas e tipos de dados.
* **Swagger:** Documentação interativa da API.

## Estrutura de Branches
O projeto utiliza branches para separar funcionalidades e implementações:
* `main`: Versão estável com persistência em memória.
* `feature/postgres` (Opcional): Branch com integração ao **PostgreSQL** para persistência de dados.
